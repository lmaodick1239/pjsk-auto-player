# PJSK Auto Player — 路线图 (Roadmap)

> **目标**: 从自动化脚本 → 带自动更新的一站式傻瓜游戏助手
>
> 当前版本: v5.7.1

---

插上手机 → 打开软件 → 自动运行。零配置、零操心、自动更新。

```

└─────────────────────────────────────────────┘
```

## 📊 现状盘点

| 维度 | 状态 | 备注 |
|------|------|------|
| 自动执行 | ✅ 成熟 | 预测引擎 + 4 种模式 |
| 异常恢复 | ✅ v5.5 | 5 级状态机 + 弹窗处理 |
| 行为多样性 | ✅ v5.6 | Session Fingerprint + 高斯抖动 |
| CI/CD 构建 | ✅ | tag 触发 → GitHub Release |
| **自动更新** | ❌ 缺失 | 用户需要手动下载新版 |
| **傻瓜式引导** | ⚠️ 基础 | 向导存在但不够智能 |
| **一键体验** | ⚠️ 部分 | 需手动配 ADB / 选设备 |
| **生态功能** | ❌ 缺失 | 体力/活动/卡池等辅助 |
| **原生桌面体验** | ⚠️ 基础 | tkinter GUI，功能较简 |

## 🗺️ 路线图

### Phase 1 (v5.8) — 自动更新系统

**目标**: 用户再也不用手动下载 GitHub Release，软件自己搞定。

| 任务 | 描述 | 预估 |
|------|------|------|
| **1.1 版本比较引擎** | 启动时异步请求 GitHub API，比对当前版本与 latest release | 1d |
| **1.2 更新通知 UI** | 状态栏/托盘提示"新版本可用"，显示 changelog，提供"立即更新"/"稍后" | 1d |
| **1.3 下载 + 校验** | 下载 Release asset，SHA256 校验完整性，断点续传 | 1d |
| **1.4 自动应用更新** | macOS: 替换 .app → 重启; Win: 替换 exe; Linux: 替换 binary | 1d |
| **1.5 配置"自动更新"开关** | 设置页 + config.yaml 控制：auto/notify-only/off | 0.5d |
| **1.6 更新进度 UI** | 下载进度条 + 速度显示 + 失败重试 | 0.5d |

**交付**: 新版本发布后，用户打开软件自动弹窗 → 点"更新" → 等几十秒 → 自动重启用上新版。

```
v5.8 用户流程:
开机 → 启动 PJSK Auto Player
    → 后台检查 GitHub Release
    → 发现 v5.9.0 可用
    → 弹窗: "发现新版本，更新? (v5.8.0 → v5.9.0)"
    → 用户点"更新"
    → 下载进度: ████████░░ 80%
    → 下载完成 → 自动替换 → 重启
    → 欢迎使用 v5.9.0!
```

### Phase 2 (v5.9) — 傻瓜化体验

**目标**: 从"能用"到"好用"，让非技术用户也能 3 分钟内上手。

| 任务 | 描述 | 预估 |
|------|------|------|
| **2.1 一键设备检测** | 启动时自动扫描 ADB 设备，无设备时弹出引导图 | 1d |
| **2.2 智能场景预设** | 根据检测到的服务器/设备推荐最优配置 (分辨率/后端/模式) | 1d |
| **2.3 首次运行智能向导** | 3 步走完: ①连设备 ②自动校准 ③开始运行，零手动输入 | 1d |
| **2.4 友好错误提示** | 所有错误用大白话显示 + 解决建议链接，不抛 Traceback | 0.5d |
| **2.5 系统托盘深度集成** | 托盘菜单: 开始/暂停/停止/状态/设置/更新，右键全搞定 | 1d |
| **2.6 状态面板精简版** | 只显示最重要的 3 个数字: 歌曲数/运行时间/当前模式 | 0.5d |
| **2.7 Windows 一键启动器** | 单 exe 包（PyInstaller 已就绪）+ 自动安装 scrcpy/minitouch | 1d |
| **2.8 macOS 一键启动器** | .app 包 + 自动授权引导 + ADB 自动检测 | 1d |

**交付**: 下载 → 打开 → 插手机 → 点"开始"，三步走完。

---

### Phase 3 (v6.0) — 一站式游戏助手

**目标**: 不只是自动执行工具，而是 Project Sekai 的瑞士军刀。

| 任务 | 描述 | 预估 |
|------|------|------|
| **3.1 活动日历** | 自动获取活动信息 (实时剩余时间/排名线/奖励进度) | 2d |
| **3.2 体力/火焰管理** | 实时读取体力值，自动判断是否烧火/等回体 | 2d |
| **3.3 智能活动调度** | 设置目标 (如"打到 TOP5000") → 自动计算所需场次 → 安排执行计划 | 2d |
| **3.4 奖励自动领取** | 活动结束后自动领取排名/歌榜奖励 | 1d |
| **3.5 每日任务自动化** | 自动登录 → 每日 Live → 商店兑换 → 区域对话 → 礼物领取 | 2d |
| **3.6 选曲推荐** | 根据活动类型/效率自动推荐最佳歌曲 | 1d |
| **3.7 桌面窗口精简模式** | 迷你悬浮窗 (只显示状态 + 按钮)，可置顶 | 1d |
| **3.8 自动更新增强** | 支持静默后台更新 (检测到新版本 → 自动下载 → 下次启动自动替换) | 1d |

**交付**: 连上就不用管了 — 自动打歌、自动回体、自动领奖、自动更新。

### Phase 4 (v6.1+) — 社区与生态

| 任务 | 描述 | 优先级 |
|------|------|--------|
| **4.1 配置同步** | 配置可导出导入，支持分享预设 | P3 |
| **4.2 多账号管理** | Profile 切换，不同账号独立配置 | P3 |

| **4.5 插件系统** | 社区可写 Python 插件，扩展功能 | P4 |

## ⚠️ 依赖项与风险

| 风险 | 影响 | 应对 |
|------|------|------|
| GitHub API 限流 (未认证) | 自动更新检查失败 | 使用 `gh` token 或内置 PAT |
| 游戏更新改 UI | 弹窗/场景识别失效 | Pipeline JSON 热修复 |
| 游戏增设反自动化措施 | 执行被封 | 行为多样性持续迭代 |
| ADB/平台差异 | 部分功能不可用 | 优雅降级 + 明确提示 |
